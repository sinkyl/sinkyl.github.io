perf(graph-api): FxHashMap + edge adjacency cache — O(k) incremental cycle check
Context: graph-api used std HashMap/HashSet (SipHash) for all internal maps
and rebuilt the full combinational adjacency O(E) on every edge connect.

Why: SipHash is ~3-5x slower than FxHash for small integer keys (NodeId,
PortId, SlotIdx). The full adjacency rebuild makes incremental cycle
detection scale with total edges rather than reachable subgraph.

Changes:
- Switch all graph-api HashMaps/HashSets to rustc_hash FxHashMap/FxHashSet
- Add edge adjacency cache (edges_from_cache, edges_to_cache) with
  generation-based staleness tracking, following children_cache pattern
- Rewrite would_create_illegal_cycle() to do lazy DFS over cached edges:
  O(k) where k = reachable subgraph, vs O(E) for full adjacency rebuild
- Bump generation on edge connect/disconnect for correct cache invalidation
- Add edges_from_cached(), edges_to_cached(), edges_of_cached() query API

Impact: faster compile-time map operations across the board; incremental
cycle check now scales with local graph topology, not total edge count.

  functions (avoids &self vs &mut self borrow conflicts)
- Rewrite settle() from &self to &mut self with worklist convergence
- Rewrite tick() convergence loop with same pattern
- Update reset_all() to clear cached dirty infrastructure
- Add nested-context benchmark helpers and benchmark suite
- Update ~30 callsites: let compiled → let mut compiled

feat(graph-api,builtins): add execution system — context plans, event-driven, batch, parallel, boundary sync
Build the full execution pipeline for heterogeneous context graphs.

Context:
The compiler previously only supported flat oneshot/ticked evaluation. Contexts
with different EvalSemantics (EventDriven, Batch) had no execution path, and
non-Alias boundary ports were never drained — callers had to manually mark
internal slots dirty.

Changes:
- ContextPlan with phased execution (DrainBoundary → Settle/DirtySettle → Latch →
  ReSettle → FlushBoundary) driven by EvalSemantics
- Event-driven execution: DirtySet, DirtyWorklist, dirty_settle with O(k) propagation
- Batch execution: scatter/gather across lane-cloned signal tables (seq + rayon parallel)
- Parallel context execution: rayon::scope with ParallelTableGuard for disjoint-slot safety
- Boundary sync: both IN (drain external→internal) and OUT (flush internal→external)
  for non-Alias contexts; settle() convergence loop drain→settle→flush→settle
- SignalTable extensions: DirtySet, BufferTable (FIFO + Streaming), Worklist, ParallelTableGuard
- ContextExecutor trait + ExecutorRegistry for GPU/DSP backend extension
- BoundarySyncMode::boundary_sync() factory on ContextPolicy
- Context presets (ContextCatalog) with full palette integration
- Builtins: graph_helpers, simplified instantiate/prefab via catalog
