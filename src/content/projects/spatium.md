A **spatial computing platform** for immersive 3D experiences, AR/VR applications, and next-generation human-computer interaction.

---

## The Problem

Building for spatial computing is fragmented. Each headset has different APIs, tracking systems, and quirks. Developers waste time on device-specific code instead of creating experiences. Collaboration in 3D spaces remains clunky.

---

## The Solution

![Mixed Reality View](/images/projects/spatium/mixed-reality.svg)
*Hand tracking and spatial anchoring across devices*

---

## How It Works

A declarative API abstracts device differences. Write once, deploy to Quest, Vision Pro, and HoloLens. The engine handles spatial anchoring, occlusion, and hand tracking with sub-millisecond latency.

![Cross-Platform SDK](/images/projects/spatium/sdk-diagram.svg)
*Unified SDK for all major XR platforms*

---

## Key Features

- **Cross-platform** — one codebase, all headsets
- **Hand tracking** — gesture recognition built-in
- **Spatial audio** — 3D sound with room acoustics
- **Collaborative** — real-time multi-user spaces
- **Asset pipeline** — glTF, USD, automatic LOD

---

## Who It's For

XR developers tired of platform fragmentation. Studios building collaborative experiences. Enterprises deploying training simulations across mixed device fleets.

---

## Current Status

Beta phase. Core SDK stable. Currently optimizing for Vision Pro and building the collaborative editing system for shared 3D workspaces.
